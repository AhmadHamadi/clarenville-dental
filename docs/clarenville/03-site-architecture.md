# Site Architecture

## Homepage Role
The homepage should function as:
- the main clinic/entity page
- the strongest local trust page
- the main Clarenville relevance page
- a dispatcher to top commercial treatment pages

It should not try to fully rank for every treatment.

## Recommended Primary Navigation
- Home
- Services
- Emergency
- New Patients
- Insurance / CDCP
- Meet the Team
- Contact / Book

## Services Hub Role
The services hub should:
- group services clearly by patient need
- include a jump menu or service selector
- give short summaries only
- push users into dedicated commercial pages

It should not duplicate full sales copy from service pages.

## Priority Page Clusters

### High-Value Commercial
- Emergency Dentist
- Invisalign
- Dental Implants
- Family / General Dentist
- Dental Cleanings
- Wisdom Teeth Removal
- Cosmetic Dentistry

### Trust / Support
- About
- Meet the Team
- New Patients
- Insurance / CDCP
- FAQ
- Contact

## Conversion Pathways

### Emergency Path
Homepage -> Emergency page -> call button -> contact page fallback

### New Patient Path
Homepage -> New Patients -> Family / Cleaning / Insurance -> Contact

### High-Intent Treatment Path
Homepage or Services hub -> Invisalign / Implants / Wisdom Teeth / Cosmetic -> Contact / Book

## Blog Strategy
For this market, blog content should be light and strategic:
- emergency symptom explainers
- practical patient education
- local trust content

The blog should support money pages, not compete with them.

## Local Area Page Guidance
Do not build multiple nearby-town pages by default.

Only justify an area page if:
- there is a real service angle
- the page helps users
- the page can contain unique local value
- it does not duplicate Clarenville-first intent
